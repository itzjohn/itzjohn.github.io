Partitioning before installation

This is best practice for installing Linux distro.

  • /boot partition if you using a Hard Drive to help speed up to boot process since this partition will be at the very beginning so it doesn’t have to search for it on the hard drive disk. 512MB is fine. This is not required for Solid State Drives.
  • EFI or EPS partition may be needed depending on the OS, usually requires 100MB. It stores Boot loaders, Device drivers, System utilities, and some key Data files that may be needed to boot your OS
  • / (root) partition is where the Linux Distro will be installed. Making it easy for you in upgrade or switch distros without losing all your personal data. 10 GB of storage if you don’t plan on installing huge amount of software. 25GB if you do, really depends on your personal choice.
  • Swap area partition for when you run out of ram or when the computer goes into sleep/hibernation to store temporary data. At least 4GB, or the same amount as your ram size. Greater than 8GB, you can keep it around 8GB.
  • /home partition is where all your personal data is stored. Uses the remaining storage you have left for this partition

Install an AppImage package

You need to give the AppImage the correct permissions. Open the terminal and type the following command
chmod u+x [Application_name].AppImage

Install a DEB package

You can either double click it and it will open up in a software installer or you can install it via terminal
sudo dpkg -i [source location of the deb package]
If the package does not depend on any other packages, then the installation it will install automatically. However, if it does you will need to install the those package using the below command, after installing your DEB package.
sudo apt -f install

Change default sound output

Open the sound settings, see what types of output devices you have available.
Play some music to figure out which output device listed in your sounds settings corresponds to which of your physical device (Speakers, headphones, monitor, etc).
Please note more than one listed output device can correspond to one physical device. For example, my USB powered speakers has two different output devices listed in the sounds settings. Digital Output and Line Out.

Write down the output device, for this example Family 17h ...

Open up the terminal and type the below command to see what cards are available.
pacmd list-cards

Here the result of my system, I omitted a lot of stuff so we can focus on Lines 1, 8, 14
Each index is a different physical device, look at the product name and active profile.

Located the following file /etc/pulse/default.pa and add the following towards the bottom of the file.
Enter the following set-card-profile [index number] [active profile] & set-default-sink [index number]

Fixing Network drive with read-only system.

sudo ntfsfix dev/sdb1

Auto mount network drives

sudo fdisk -l to see a list of all the drives.
sudo edit /etc/fstab opens up a file which you input the drive details for auto mount.

/dev/sdb1 /media/itzjohn/Network ntfs defaults 0 0

Hide windows partition

sudo fdisk -l to see a list of all the drives.
sudo edit /etc/fstab opens up a file which you input the drive for auto mount.

/dev/disk/by-uuid/50886740886723A6 /mnt/50886740886723A6 auto nosuid,nodev,nofail,noauto 0 0
